# Guía de commits

Este documento resume el flujo de trabajo utilizado en el repositorio.

## Ramas utilizadas

- `main`: rama principal del proyecto.
- `dev`: rama de integración.
- `Avances_de_Pablo`: rama utilizada para avances de modelamiento, algoritmo, datos y visualización.
- `Avances_de_Bernardo`: rama utilizada para avances de interfaz, documentación y pruebas.

## Criterio de commits

Los commits deben describir claramente el cambio realizado. Por ejemplo:

- `Agregar carga de grafo desde CSV`
- `Implementar algoritmo de Dijkstra`
- `Agregar interfaz gráfica`
- `Actualizar informe técnico`
- `Corregir documentación de fuentes`

## Flujo de trabajo

1. Trabajar en una rama propia.
2. Guardar cambios con commits descriptivos.
3. Subir la rama a GitHub.
4. Integrar los cambios mediante merge o Pull Request.
5. Verificar que la aplicación ejecute correctamente antes de fusionar a `main`.
